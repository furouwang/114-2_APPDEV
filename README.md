## 課程說明

本課程引導學生理解 Android APP 開發的完整工作流程，涵蓋 Java 物件導向程式語言、Android Studio 開發工具及雲端資料庫整合等核心技術。課程同時導入跨平臺開發工具 React Native，使學生具備同時開發 Android 和 iOS APP 的能力，並融入物聯網和 AI 應用等技術主題。

課程採用專題式學習，學生將透過分組合作完成 APP 專題，在實作過程中提升程式設計、團隊溝通與專案管理能力。

**教學目標：**

1. 提高 APP 專題的功能完整度
2. 提高 APP 專題的功能使用性
3. 提高 APP 專題的功能穩定性

## 教學方法

本課程採用專題式學習 Project-Based Learning 結合小組合作學習，並融入 AI 科技輔助教學。參考 Xiaoqi et al. (2024) 所提出的 AI 支援 PBL 實施框架，課程涵蓋六個階段：

| 階段 | 對應週次 | 說明 |
|------|---------|------|
| 專案識別 | 第 1-3 週 | 課程介紹、環境建置、需求分析 |
| 計畫發展 | 第 4-5 週 | 專案管理、工作分配、介面設計 |
| 活動探索 | 第 6-9 週 | 核心技術學習、功能開發、程式碼審查 |
| 作品產出 | 第 10-12 週 | AI 輔助開發、功能整合、系統測試 |
| 結果溝通 | 第 13-15 週 | 發布、優化、安全性實作 |
| 活動評估 | 第 16-18 週 | 成果整理、展示準備、期末發表 |

全期程皆有小組合作學習活動，第 10 週起導入 AI 工具輔助開發。

## 作業繳交方式：Fork + Pull Request

本課程採用業界標準的 Fork + Pull Request 流程進行作業繳交與管理。

```
教師建立課程 Repo
       ↓
學生 Fork 到自己的帳號（學期初做一次）
       ↓
每週在 Fork 中建立 weekXX/ 資料夾，完成作業
       ↓
git add → commit → push 到自己的 Fork
       ↓
到 GitHub 網頁發 Pull Request 回教師 Repo
       ↓
教師在 PR 列表查看繳交狀況並給予回饋
```

### 初始設定（學期初做一次）

1. 到教師的課程 Repo 頁面，點擊右上角 **Fork**
2. Fork 完成後，Clone 到本機：
   ```bash
   git clone https://github.com/你的帳號/114-1-appdev.git
   cd 114-1-appdev
   ```

### 每週作業繳交流程

1. 在 Fork 中建立當週資料夾，完成作業
   ```bash
   mkdir week03
   # 在 week03/ 中完成作業
   ```
2. 提交並推送
   ```bash
   git add .
   git commit -m "完成第三週作業"
   git push origin main
   ```
3. 到 GitHub 網頁，點擊 **Contribute > Open pull request**
4. 確認 PR 標題格式正確後送出

### PR 標題格式

```
學號_姓名_weekXX
```

範例：`C11200001_王小明_week03`

> PR 本身即為繳交紀錄，包含時間戳記和修改歷程。教師會直接在 PR 留言給予回饋與成績。

## 專題製作

課程採用雙 Repo 架構管理每週作業與小組專題：

| Repo | 用途 | 操作方式 |
|------|------|---------|
| 課程 Repo | 每週作業繳交 | 學生 Fork 後發 PR |
| 專題 Repo | 小組專題開發 | 組長 Use this template 建立 |

### 分組與選題流程

| 階段 | 時間點 | 說明 |
|------|--------|------|
| 分組與公布題目 | 第 3 週 | 公布分組名單和題目參考清單，組長建立專題 Repo |
| 個人題目探索 | 第 4 週 | 每人提出 1-3 個題目構想 |
| 決定題目與提案 | 第 5 週 | 組內討論投票，繳交提案 |
| 提案審查 | 第 6 週 | 教師審查並提供建議 |

### 組長建立專題 Repo 步驟

1. 到教師的專題模板 Repo，點擊 **Use this template > Create a new repository**
2. 命名為 `114-1-appdev-G組別`，例如：`114-1-appdev-G01`
3. 設為 **Public**
4. 進入 **Settings > Collaborators**，加入組員、老師和助教，權限設為 Write
5. 被邀請的人需到 GitHub 通知點 **Accept invitation**

### 專題 Repo 結構

```
114-1-appdev-G01/
├── README.md              ← 專題說明與進度追蹤
├── .gitignore
├── proposal/              ← 專題提案
│   └── proposal.md
├── my-topics/             ← 個人題目探索
│   └── topic_template.md
├── src/                   ← 程式碼
│   ├── app/
│   └── utils/
├── docs/                  ← 報告與投影片
│   ├── report.md
│   └── slides/
└── demo/                  ← APP 操作影片與截圖
```

## 成績評量

### 成績比例

| 項目 | 比例 | 說明 |
|------|------|------|
| 期中報告 | 30% | 小組上台報告專題提案，繳交提案報告與投影片 |
| 期末報告 | 30% | 小組上台報告結案，展示 APP 操作或預製影片，繳交結案報告與投影片 |
| 小考 | 20% | 共 4 次，分別於第 4、7、11、13 週 |
| 作業繳交 | 10% | 課堂練習程式範例，透過 Fork + PR 繳交 |
| 平時成績 | 10% | 出席記錄、課堂提問與討論 |

### 專題評分架構

| 維度 | 評量對象 | 配分 | 評分方式 |
|------|---------|------|---------|
| 教師評分 | 整組 | 40% | 教師依提案品質、程式碼品質、Demo 表現等細項給分 |
| 組間互評 | 整組 | 20% | 期末發表時各組互評 |
| 組內互評 | 個人 | 20% | 匿名評分，評估貢獻度與合作能力 |
| AI 數據分析 | 個人 | 20% | 分析 GitHub commit 次數、時間分布、程式碼貢獻等數據 |

### 同儕互評說明

**組間互評**：期末發表時，每組對其他組評分，每項 1-5 分，評估項目包含簡報表達、技術難度、實用價值、整體完成度。

**組內互評**：每人有 100 分分配給其他組員，評估工作投入、技術貢獻、團隊合作。需提供具體事例支持評分。

## Git 快速指南

### 常用指令

```bash
# 複製遠端 Repo 到本機
git clone https://github.com/你的帳號/repo名稱.git

# 查看目前狀態
git status

# 將所有變更加入暫存區
git add .

# 提交變更
git commit -m "簡述這次修改的內容"

# 推送到 GitHub
git push origin main

# 從遠端拉取最新內容
git pull origin main
```

### 注意事項

- 每次作業都要先 `git pull` 確保本機是最新狀態
- Commit 訊息請用中文或英文簡述修改內容，避免使用無意義訊息
- 請勿將大型檔案（如 .apk、影片素材）直接 push 到 Repo
- 遇到 merge conflict 時，請先嘗試解決，無法解決可於課堂請教
